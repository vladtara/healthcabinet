## What Went Well

1. **"Reskin only" discipline held perfectly** — Zero behavior regressions across 3 reskin stories (13-1, 13-2, 13-3). Scripts untouched except imports. All queries, mutations, navigation targets preserved exactly.

2. **DataTable primitive proved reusable** — 13-2 established the pattern (column definition, `children` snippet, `onRowClick`), 13-3 copied it verbatim. Primitive adoption over custom markup.

3. **ConfirmDialog adoption fulfilled retro Action Item 1** — User suspension in 13-3 was the first admin flow to use the extracted primitive. Focus trap, Escape, backdrop-click, loading state all inherited.

4. **Import-guard regex became project-wide** — 7 test files enforce shadcn primitive exclusion with terminator regex. No regressions possible without test failure.

5. **CSS section-prefix convention rock-solid** — ~102 new classes across 5 stories, all following `.hc-[section]-*` naming with zero drift.

6. **Frontend redesign initiative COMPLETE** — 7 epics (Epics 7-13), all done. Every major route reskinned from Tailwind to 98.css Windows 98 clinical workstation aesthetic.

7. **Nested `<main>` landmark fixed app-wide** — 9 page components fixed across 13-4 (admin) and 13-5 (app). HTML5 landmark violation eliminated.

---

## What Didn't Go Well

1. **Cross-functional action items never got stories** — 3 of 5 Epic 12 retro action items were not addressed because they required PM, architect, or tech-writer involvement and no story was created for them.

2. **SSE security deferred for the 4th time** — Flagged in Epic 11, carried through Epic 12, explicitly scheduled for 13-5, re-deferred with GA-waiver. Frontend-only stories cannot fix a frontend+backend problem.

3. **Docker unavailability silently skipped test verification** — Stories 13-4 and 13-5 completed without running tests in Docker Compose. Test counts are approximate. Desktop QA visual check never executed.

4. **3 pre-existing test failures persisted the entire epic** — `documents/page.test.ts`, `AIChatWindow.test.ts`, `users.test.ts` were "out of scope" in all 5 stories. Nobody owned them.

5. **9 Tailwind component files remain un-migrated** — The "final" hardening story (13-5) documented but didn't reskin: `AiInterpretationCard`, `AiFollowUpChat`, `PatternCard`, `AIClinicalNote`, `BiomarkerTrendChart`, `BiomarkerTrendSection`, `HealthValueBadge`, `PatternAlertSection`, `dashboard/+page.svelte`.

---

## Key Insights

1. **The system produces stories well but doesn't produce cross-functional coordination tasks.** Retro action items that need multiple roles (PM + architect, dev + backend) must become real stories in sprint-status with explicit tasks — not just bullet points in a retro doc.

2. **"Deferred with rationale" is documentation, not resolution.** Four deferrals on the same SSE item proves that rationale-quality doesn't correlate with action. The scheduling mechanism needs to change.

3. **Docker availability is a hard dependency.** Treat `docker compose exec` failure as a HALT condition. Notify DUDE immediately. Never proceed with "changes are mechanical" rationalization.

4. **Primitive adoption > custom markup.** DataTable and ConfirmDialog adoption in 13-2/13-3 reduced per-story effort and eliminated entire categories of bugs (keyboard navigation, focus traps, sort state).
