## What Went Well

1. **Pattern reuse compounded velocity** — Stories 12-3 and 12-5 shipped with **zero** code-review patches because 12-1/12-2/12-4 had already locked down the CSS prefix convention, fieldset/legend pattern, state banner usage, and test shape. Five stories landed on a single day (2026-04-05).

2. **First destructive-action dialog, built right** — Story 12-4 established the 98.css dialog contract: `role="dialog"` + `aria-modal`, focus trap with Tab cycling + `requestAnimationFrame` programmatic focus, Escape + backdrop close, type-to-confirm email gate. Review patches caught focus trap and Escape handler placement before merge.

3. **Regulatory compliance handled at the schema level** — Migration 013 flipped `consent_logs.user_id` FK from CASCADE to SET NULL + nullable, so GDPR Article 17 (right to erasure) works without violating consent-log retention requirements. Audit logs get `user_id` redacted before user row deletion. This is the kind of decision that's painful to retrofit — good it landed here.

4. **Zero regressions across 520 tests** — Test count grew +20 frontend and +6 backend with 0 failures at every checkpoint. Axe audits passed on every story. The "reskin, don't rewrite" discipline from Epic 11 held.

5. **Reusable form pattern shipped as a bonus** — Story 12-5's dirty-state + navigation-guard pattern is directly reusable for any future form (admin edit pages in Epic 13). It's a project asset, not just a 12-5 feature.

---

## What Didn't Go Well

1. **Inline dialog markup is a DRY debt** — 12-4's confirmation dialog lives inside `settings/+page.svelte`. Epic 13 will need admin confirmations (user ban, bulk corrections). Either extract now, or face copy-paste divergence. Flagged at planning time ("build as inline markup... not a separate component") but deserves a second look.

2. **Deferrals from earlier epics still deferred** — Epic 11 retro's action items #2 (`HealthValueRow` Tailwind migration) and #3 (SSE `EventSource` token-as-query-param security) were NOT addressed in Epic 12. Both are out of scope, but the debt is growing. Epic 13-5 (frontend hardening) is the last chance before shipping.

3. **MinIO orphan objects after account deletion** — 12-4 backend deletes DB rows but not MinIO objects. Deferred to Story 6.2 scope. A user who exercises right-to-erasure today leaves document blobs in object storage. Regulatory risk.

4. **Pre-existing test infra fragility acknowledged repeatedly** — `mockIsPending` closure capture and `?raw` import bundler fragility (both in 12-1) flagged in review but deferred. Theoretical, but it's the second time they surface.

5. **Admin RESTRICT FK edge case unresolved** — If an admin user attempts self-deletion, `audit_logs.admin_user_id` RESTRICT FK may block it. 12-4 marked it an edge case and deferred. Relevant for Epic 13 (admin management surface).
