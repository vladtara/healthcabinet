## Intent

**Problem:** AI-generated content (dashboard Clinical Note, chat replies, cross-upload patterns) is always produced in English regardless of the user's selected locale. Additionally, the dashboard Clinical Note does not reliably refresh after a new document is processed — a 429 rate-limit hit combined with `retry: false` silently leaves the note stale for the rest of the rate-limit window when multiple uploads complete in quick succession.

**Approach:** Thread the frontend `localeStore.locale` value to four AI backend endpoints via a `locale` query/body param; append a language-directive line to three prompt templates (dashboard interpretation, follow-up chat, patterns); localize fallback and disclaimer strings that are streamed back to the user. Fix the stale-note bug by adding `staleTime: 60_000` to the dashboard interpretation query and relaxing `retry: false` to allow one retry on error. Storage policy is option (c): existing per-document `AiMemory` rows are not touched.

## Boundaries & Constraints

**Always:**
- The internal locale codes are `'en'` and `'uk'`; the backend `locale` param accepts the same values and defaults to `'en'`.
- Existing per-document `AiMemory` interpretations (generated by the processing pipeline) are NOT regenerated — only freshly generated content uses the locale. No DB migration.
- `INTERPRETATION_PROMPT_TEMPLATE` (processing pipeline) is out of scope and must not be changed.
- Rate limit constants and safety forbidden-pattern list (`safety.py`) are not changed.

**Ask First:**
- If any existing AI endpoint integration test asserts the exact prompt text or response shape, halt before patching that test and confirm the new assertion.

**Never:**
- Do not add a `preferred_locale` column to the users table or any other DB table.
- Do not regenerate stored `AiMemory` rows from any migration, cron job, or service path.
- Do not change `staleTime` or retry settings for the `documents`, `health_values`, or `baseline` queries — only the `ai_dashboard_interpretation` query.

| Scenario | Input / State | Expected Output / Behavior | Error Handling |

## Design Notes

**Language instruction placement:** Each prompt ends with the language directive as the final bullet/sentence (e.g. `"- Respond in Ukrainian."`) so it overrides any implicit language the model might infer from the context data. The instruction is intentionally terse.

**Fallback strings as locale dicts:** The four fallback string constants become two-entry dicts rather than computed strings, so the locale lookup is a simple `dict.get(locale, dict['en'])` — no branching in the hot path and easy to extend if more locales are added.

**staleTime = 60 s rationale:** The dashboard rate limit is 10 calls per 60 s. Adding `staleTime: 60_000` means at most one refetch per window naturally, so a ten-document batch upload never hits the ceiling. The single `retry: 1, retryDelay: 2000` handles a transient 429 on the first post-upload refetch without permanently suppressing the note.
