# Epic 11 Retrospective — Documents Cabinet & Upload Workflow

**Date:** 2026-04-05
**Facilitator:** Bob (Scrum Master)
**Epic:** 11 — Documents Cabinet & Upload Workflow (98.css Frontend Redesign)
**Stories:** 5/5 completed (100%)

---

## What Went Well

1. **Pattern reuse accelerated velocity** — Story 11-1 established CSS class conventions (`.hc-[section]-*`), 98.css integration, and test approaches. Stories 11-3 through 11-5 reused these patterns and shipped faster.

2. **Zero regressions across 5 stories** — Test count grew from 377 to 483 (+106 net new) with 0 failures at every checkpoint. "Reskin not rewrite" kept existing behavior stable throughout.

3. **Code review caught real issues** — 12 patches applied across the epic: focus traps, missing default cases, wrong ARIA roles, button hover mismatches, missing tests, optimistic flag rollback. Each round improved quality.

4. **Visual alignment course correction** — Mid-epic comparison with design mockup (`ux-design-directions-v2.html`) caught the WindowFrame vs section header mismatch. Fixed the Import page, DocumentUploadZone, and ProcessingPipeline to match the mockup before the epic closed.

5. **Consistent CSS architecture** — 40+ new CSS classes organized by feature with clear `.hc-[section]-*` naming: `.hc-doc-*`, `.hc-detail-*`, `.hc-import-*`, `.hc-pipeline-*`, `.hc-recovery-*`. No scoped styles, no Tailwind structural classes in new code.

## What Didn't Go Well

1. **Initial design drift on Import page** — Story 11-3 shipped with `WindowFrame` (blue Win98 title bar) when the mockup showed a section header pattern. Required rework. Root cause: design mockup not consulted early enough.

2. **Pre-existing Tailwind debt in child components** — `HealthValueRow` and `PartialExtractionCard` had internal Tailwind classes that were flagged in every review. Story 11-5 fixed PartialExtractionCard, but HealthValueRow's card layout was only replaced with inline table rows in the detail panel — the component itself still has Tailwind.

3. **SSE token security concern deferred 3 times** — Token passed as query param to `EventSource` flagged in reviews for stories 11-1, 11-2, and 11-3. Deferred each time as "pre-existing." Growing technical debt.
