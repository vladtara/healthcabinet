## What Went Well

1. **SSE token-in-URL security finally shipped after 4 deferrals** — First flagged Epic 11 retro, re-deferred in Epic 12 retro, re-deferred in 13-5 with GA-waiver, re-deferred in Epic 13 retro. Story 14-1 delivered: backend dual-auth (Authorization header preferred, query-param fallback), frontend fetch-based `streamDocumentStatus()` utility, explicit `AbortController` lifecycle management, two consumers migrated (`ProcessingPipeline.svelte`, `documents/+page.svelte`). **The single biggest follow-through win of the redesign initiative.**

2. **Cross-functional discipline held** — For the first time, the systemic fix from Epic 13 retro ("cross-functional items become real stories") was applied uniformly. Three items that had been retro bullets for 3+ epics (SSE security, MinIO orphan cleanup, admin self-deletion ADR) each got a dedicated story with explicit backend AC and cross-layer task breakdown.

3. **Prefix-based bulk deletion pattern** — Story 14-2 switched account-level MinIO cleanup from per-key decrypt-then-delete (brittle, per-record failures cascade) to `{user_id}/` prefix sweep (simple, catches orphans from prior failed cleanups, idempotent). Directly enables Story 6-2's GDPR Article 17 cascade without re-architecture.

4. **Dual-auth zero-downtime migration** — Story 14-1's dual-auth approach (header preferred, query-param fallback) supports mid-stream clients during deploy, with explicit sunset planning. Reusable pattern for any auth-scheme sunset.

5. **ADR discipline restored as primary deliverable** — Story 14-3 treated the admin self-deletion ADR as the *primary* output (not a side effect of the migration). SET NULL precedent cited from Migration 013 (`consent_logs.user_id`) and `health_values.flag_reviewed_by_admin_id`; service-layer explicit redaction as defense-in-depth over FK `ondelete=SET NULL`.

6. **Test baseline fully green for the first time since Epic 10** — Story 14-4 delivered 363 backend + 578 frontend passes with 0 failures and 0 known-broken files. The 3 test files that had been "out of scope" for 5 straight stories in Epic 13 are now owned and working.

7. **Desktop QA verified across all three widths** — 10 routes (5 app + 5 admin) verified at 1024/1440/2560px with no layout breaks. Only 2 cosmetic issues deferred (no max-width at 2560px, admin stat card label wrap at 1440px).

8. **Rate of new deferrals fell sharply** — 5 new items added to `deferred-work.md` during Epic 14 (vs. +16 in Epic 12, +18 in Epic 13 per the committed file). The "cross-functional items get stories, not bullets" discipline plus fewer frontend-only epics appears to be slowing new-debt creation.

## What Didn't Go Well

1. **Docker HALT triggered late, not preventatively** — Stories 14-1, 14-2, 14-3 shipped without running tests in Docker (re-used local verification patterns). Story 14-4 caught `.dockerignore` excluding `tests/` and rebuilt the image, but by then three stories' worth of changes lacked Docker-verified baselines. The HALT rule worked *once triggered*; earlier enforcement (pre-flight check at story start, not at test run) would have caught it in 14-1.

2. **Silent backend behavior change slipped into 14-2** — The account-deletion flow included an unannounced change to consent API behavior, caught only in code review (commit `29319d0`: "remove duplicate MinIO cleanup, shield anti-pattern, and silent consent API change"). Exactly the failure mode Epic 13 warned about — backend changes without cross-functional visibility. Story scoping labelled this "backend only" but it was frontend-adjacent.

3. **Review patches pattern persists** — 7 patches across 5 stories, similar ratio to Epic 13 (12 across 5). Recurring theme: architectural constraints that belong in AC Dev Notes get discovered during review instead:
   - 14-1: fetch-based SSE dies permanently after single disconnect or 120s timeout; `auth-error` not handled by either consumer; malformed Authorization header falls back to query-param (fails open); double-click race on reconnect.
   - 14-2: `asyncio.shield` re-raises `CancelledError` and defeats its own purpose; no durable reconciliation for racing upload/reupload during deletion; duplicate inline + deferred MinIO cleanup.
   - 14-3: ADR file missing initially; stale service comment after admin_id redaction; inconsistent scalar accessor between retention tests.

4. **9 Tailwind components re-deferred without escalation** — Story 14-5 produced the inventory (per-file complexity ratings; `.hc-skeleton-*` as migration target) as scoped, but the *decision of when Tier 2 migration ships vs. Epic 6 scope* has no owner. Bonus finding: `AiFollowUpChat.svelte` and `dashboard/+page.svelte` still import shadcn `Button`/`Textarea` primitives that should have been removed in Epic 7. This is Epic 7 bleed-through, not just Tailwind debt.

5. **`deferred-work.md` has grown every epic of the redesign track, including this one** — Bullet-item count in the committed file: 129 at Epic 12 start → 145 at Epic 13 start (+16) → 163 at Epic 14 start (+18) → 164 at Epic 14 end (+1 after removing 4 SSE items that 14-1 resolved). The rate of addition is falling, but **the total is still monotonically increasing.** Worse, the file is not maintained when stories close items — SSE security and SSE reconnect were shipped by 14-1 days ago but were still in `deferred-work.md` until this retro's cleanup pass. There is no Definition of Done rule that says "if your story resolves a deferred item, remove it from the file." Multi-deferred items still open: admin destructive-action double-click guard (3×).

6. **Initial retro metrics were narrative, not measured** — The first draft of this retro claimed "6 resolved vs 5 new, net shrinking for 3rd consecutive epic." DUDE pushed back; actual `git diff` against the file showed +1 net and 4 items still-listed-but-resolved. Lesson: retro metrics for deferred work must be computed from the file (`git diff`, `wc -l`), not from story narrative. Same issue applies retroactively to Epic 13 retro's "10 resolved vs 9 new" claim — the file actually grew by 18 during Epic 13.

6. **Story 14-1 was not test-verified before merge review** — The Docker outage prevented `npm run test:unit` and `pytest` runs during 14-1 dev. Tests were written but not executed until 14-4. No regressions materialised, but the audit trail is weaker.

## Key Insights

1. **Retro action items work when they become stories, not when they stay bullets.** Epic 14 closed 5/5 Epic 13 commitments because each became a numbered story with AC, owner, and sprint-status entry. The three prior epics that kept deferring SSE security all tried to resolve it via "retro bullet + GA-waiver" and failed. The systemic fix is structural, not motivational.

2. **Defense-in-depth beats framework-behavior-only.** FK `ondelete=SET NULL` + explicit service-layer `admin_id=NULL` statement (14-3) makes the intent legible without relying on ORM internals. Fire-and-forget MinIO cleanup in a deferred ARQ job + structured logging (14-2) survives request cancellation without `asyncio.shield` anti-patterns. Fetch-based SSE + explicit `AbortController` cleanup on unmount + event-driven error close (14-1) replaces `EventSource` auto-reconnect implicit behavior.

3. **Tests reveal real component issues, not "test problems."** Fixing the 3 broken test files in 14-4 required: exposing a missing close button on `DocumentDetailPanel`, raising its z-index above the header background, replacing a `<dialog>` element with `<section role="region">` to match actual DOM, adding a `role="textbox"` ARIA label to `AIChatWindow`'s contenteditable, and providing `ReadableStream` in the jsdom `Response` mock. "The test is broken" almost always means "the component is broken and nobody noticed."

4. **Architectural constraints must live in Dev Notes, not review comments.** ~3 of the 7 code-review patches in Epic 14 surfaced constraints that were *knowable* at story-creation time (fetch SSE has no auto-reconnect, `asyncio.shield` re-raises `CancelledError`, consent API is frontend-adjacent). A 30-min pre-implementation architecture sync (dev + architect + PM) should catch these before implementation starts.

5. **Deferred work is sticky and self-extending.** Once a path is released (query-param SSE auth), removing it requires business sign-off and coordinated client migration. Dual-auth is correct short-term; the *sunset decision* is a separate story and remains unassigned for Epic 14 AC1.2 ("remove query-param path once clients migrated").

6. **Prefix-based sweeps are the right primitive for account-scoped bulk operations.** Per-record operations (decrypt, delete each) have failure modes that compound. Prefix sweeps (`{user_id}/`) are idempotent, catch orphans from prior failed attempts, and reduce the blast radius of bugs. Applicable beyond MinIO — any storage system where the scope key is the folder prefix.
