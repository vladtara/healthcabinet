Problem
Health data exists. Meaning does not.

Patients own the files, but not the longitudinal understanding.

Current reality
  • Results arrive as PDFs, scans, screenshots, and paper printouts.
  • Each lab is interpreted as a snapshot, not a timeline.
  • Users google abnormal values and still leave uncertain.
  • Chronic-condition users repeat the same confusion every few months.
What the product must replace
Today Needed instead
Static PDF archive Patient-owned health memory
Single-result interpretation Cross-upload pattern recognition
Raw numbers and flags Plain-language, contextual meaning
Portal-by-portal fragmentation One coherent timeline

User pain is repetitive, not one-time.

The strongest use case is not a novelty upload. It is the person who keeps testing and never gets a patient-owned longitudinal view.

The opportunity is continuity.

HealthCabinet does not need to invent more health data. It needs to transform existing documents into compounding intelligence.

Trust and safety
Sensitive product, serious handling

The trust model is part of the product, not a footer afterthought.

Data and compliance posture
AWS eu-central-1 target
  • EU-region data handling posture
  • Consent, export, and deletion flows in scope
  • Encryption posture for health data and tokens
  • No casual treatment of special-category data
AI safety posture
Informational, not diagnostic
  • AI explains patterns and context
  • Uncertainty should be surfaced, not hidden
  • No treatment or dosage instruction framing
  • Human review path exists for extraction edge cases

Patient trust

Users need to believe their history is secure, exportable, and under their control.

Operational trust

Admin tools matter because document intelligence will always have edge cases in the real world.

Brand trust

The clinical UI helps, but real credibility comes from bounded claims and visible safety choices.
